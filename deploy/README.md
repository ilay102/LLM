# VIREN Pilot Deployment

This directory contains everything needed to run a 2-week shadow pilot
at a customer site.

## Scripts

| Script | Purpose |
|---|---|
| `pilot.sh` | One-command deployer: brings up gateway + redis, generates a per-client key, health-checks, runs a smoke call. |
| `teardown.sh` | Stops the stack, archives logs, preserves audit trail. |
| `daily_summary.py` | Reads the pilot's `shadow_log.jsonl` and prints (or writes CSV) cost savings, cache hit rate, tier distribution, latency. |

## 30-second quickstart

From the **repo root**:

```bash
./deploy/pilot.sh \
    --client-id acme \
    --anthropic-key sk-ant-... \
    --openai-key sk-... \
    --base-port 8000
```

You'll get:
```
╔════════════════════════════════════════════════════════════════╗
║              VIREN — Pilot deployment READY                    ║
╚════════════════════════════════════════════════════════════════╝

Client:        acme
Gateway URL:   http://localhost:8000/v1
API key:       pilot-7f3a9b2c1d4e8f0a
Logs:          deploy/clients/acme/logs/
...
```

Hand the URL + API key to the customer's engineer along with the relevant
integration sample (Python or Node — see `integration_samples/`).

## Daily progress check

```bash
./deploy/daily_summary.py --client-id acme --since 7d --csv
```

Prints a clean per-pilot report — total calls, cost saved, cache hit rate,
tier distribution, p50/p95/p99 latency, top 5 most expensive prompts.
Also writes `summary.csv` next to the log. Person B uses this for the
weekly progress email to the customer.

## End-of-pilot report

After 2 weeks, you'll use the shadow-eval harness to produce the actual
quality + cost HTML report. That's a separate command:

```bash
cd shadow-eval
python run_pairwise.py \
    --dataset deploy/clients/acme/logs/captured.jsonl \
    --out pairwise_results.jsonl
python generate_report.py --in pairwise_results.jsonl --out report.html
```

Then email the `report.html` to the customer. That's the artifact that
closes the deal.

## Teardown

```bash
./deploy/teardown.sh --client-id acme
```

Stops the stack, archives logs to a timestamped tarball. Use
`--delete-data` to also remove the Redis volume.

## Troubleshooting

**"docker daemon not running"** — start Docker Desktop, or
`sudo systemctl start docker` on Linux.

**"port already in use"** — pass `--base-port 8100` (or any free port).
The script picks Redis on `port+379` and Insight on `port+1`.

**Gateway health check times out** — usually means the customer's network
is blocking outbound to api.openai.com / api.anthropic.com. Verify with:
```bash
docker compose --env-file deploy/clients/<id>/.env logs gateway
```

**Smoke test returned unexpected content** — Anthropic/OpenAI API key
might be invalid. Test with:
```bash
curl https://api.anthropic.com/v1/messages \
    -H "x-api-key: <key>" -H "anthropic-version: 2023-06-01" \
    -H "content-type: application/json" \
    -d '{"model":"claude-haiku-4-5","max_tokens":10,"messages":[{"role":"user","content":"hi"}]}'
```

## Security notes for the customer

- All traffic stays in the customer's VPC. The gateway only talks
  outbound to the LLM providers they configured.
- Logs and cache contents live in their environment — never transmitted
  to us. They control retention.
- The pilot key (`pilot-...`) is per-pilot; revoke by tearing down.
- We can enable PII redaction at the gateway (Microsoft Presidio) on
  request; off by default for speed.
